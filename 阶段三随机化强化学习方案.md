## 当前结论

阶段三不再直接从 24 个场景全量随机训练开始，而是采用课程训练：

```text
第一阶段 easy8   ：场景 ID 0-7
第二阶段 basic14 ：场景 ID 0-13，当前默认训练课程
第三阶段 tail10  ：场景 ID 14-23，专门学习长时程尾追和高机动场景
第三阶段也可拆分为 tail4 与 hard6：

```text
tail4 -> 场景 ID 14-17，长尾追过渡场景
hard6 -> 场景 ID 18-23，高机动尾追场景
```

如果 tail4 直接训练效果不明显，先使用：

```text
tail4_warmup -> 场景 ID 14-17，但降低飞机速度、机动幅度、发射延迟和随机扰动
```

如果纯 PPO 在 tail4_warmup 仍然学不动，启用规则制导辅助：

```text
tail4_residual -> 场景 ID 14-17，最终动作 = baseline_tail_guidance * alpha + PPO_action * beta
```

默认：

```text
STAGE3_RESIDUAL_ALPHA=1.0
STAGE3_RESIDUAL_BETA=0.25
STAGE3_BASELINE_TAIL_GAIN=2.2
STAGE3_BASELINE_TAIL_OFFSET=8.0
```

其中 baseline_tail_guidance 负责给出尾追基础方向，PPO_action 只学习残差修正。
`STAGE3_BASELINE_TAIL_OFFSET` 控制规则尾追瞄准飞机尾后多远，数值更大时更偏向形成稳定尾追轨迹。

第四阶段 mix24   ：0-13 占 60%，14-23 占 40%，用于混合微调
第五阶段 full24  ：场景 ID 0-23，用于后期泛化和压力训练
```

这样做的原因是：15-24 属于长时程、高机动、尾追场景，命中时间可达 30-41 秒。如果一开始把这些场景和随机速度、随机发射延迟、随机机动幅度一起加入，PPO 很难稳定获得命中奖励，容易长期只学到低质量逼近行为。

含义：

```text
easy8   -> 随机场景 ID 0-7

tail4_residual -> 随机场景 ID 14-17，规则尾追基础动作 + PPO 残差
hard6   -> 随机场景 ID 18-23
tail10  -> 随机场景 ID 14-23
mix24   -> 60% 随机场景 ID 0-13，40% 随机场景 ID 14-23
full24  -> 随机场景 ID 0-23
```

固定单个场景可继续使用：

```text

```

## 2. 随机化模式

继续使用：

```text
STAGE3_RANDOMIZATION_MODE=train
STAGE3_RANDOMIZATION_MODE=eval
STAGE3_RANDOMIZATION_MODE=stress
```

范围：

```text
train  ：飞机速度 20-28 m/s，轻度随机化
eval   ：飞机速度 18-32 m/s，中等随机化
stress ：飞机速度 30-36 m/s，压力随机化
```

建议训练顺序：

```text
basic14 + train
basic14 + eval
tail4_warmup + train
tail4   + train
tail4_residual + train
hard6   + train
tail10  + train
mix24   + train
full24  + train
full24  + eval
full24  + stress
```

## 3. 观测空间

观测维度从 13 维扩展到 18 维。

当前观测包含：

```text
相对位置
相对速度
导弹速度方向
视线方向 LOS
目标距离
闭合速度
横向相对速度大小
导弹高度
归一化时间
飞行阶段标志
```

这样做是为了让策略更容易判断目标运动趋势，尤其是横穿、爬升、俯冲和机动规避场景。

## 4. 奖励函数

当前奖励思路：

```text
距离变近：正奖励
朝向尾部瞄准点：小正奖励
尾后对准：小正奖励
近距逼近：分段正奖励
命中：大正奖励
撞地/出界/超时：负奖励
动作过大：仅在 GUIDANCE 阶段惩罚
前出目标：仅在 GUIDANCE 阶段且距离较近时轻微惩罚
```

重点变化：

```text
命中奖励从 +10 提高到 +35
加入 10m / 5m / 2.5m 近距奖励
取消助推段动作惩罚和前出惩罚
大幅降低 ahead_penalty，避免智能体尚未接管时被强负奖励误导
```

## 5. 命中判定

Isaac Lab 环境的基础命中半径改为：

```text
hit_radius = 2.5 m
```

高机动尾追场景与阶段二有效命中半径对齐：

```text
long_weave_tail_chase       2.70 m
extended_maneuver_follow    1.70 m
climb_dive_weave_chase      1.95 m
delayed_sustained_evasion   2.45 m
19-24 高机动场景             2.55 m
```

同时命中距离改为导弹头部上一帧到当前帧的扫掠线段，与飞机尾部/机体段计算最近距离，降低高速穿越时漏判命中的概率。

## 6. PPO 配置

当前 RL-Games PPO 调整：

```text
gamma: 0.997
horizon_length: 64
minibatch_size: 1024
entropy_coef: 0.008
```

原因：

```text
任务时程较长，稍微提高折扣因子；
rollout horizon 从 32 提到 64，让一次更新看到更完整的制导片段；
minibatch_size 设为 1024，兼容 16/128 env smoke 和 4096 env 正式训练；
降低但保留探索熵，避免动作长期随机抖动。
```

## 推荐验收顺序

```text
1. easy8 smoke 能正常启动，观测维度为 18，无 NaN/inf。
2. basic14 + train 训练 300-500 iterations，观察 mean_closest_distance 是否下降。
3. basic14 + eval 可视化，确认不再明显绕圈、撞地或远离目标。
4. basic14 稳定后优先切 tail4_warmup + train，从已有 checkpoint 继续训练。
5. 若 tail4_warmup 仍然学不动，切 tail4_residual + train。
6. tail4_residual 有效果后，再逐步降低 STAGE3_RESIDUAL_ALPHA、提高 STAGE3_RESIDUAL_BETA。
7. residual 版本稳定后再切 tail4 或 hard6 + train。
8. tail4/hard6 都有进展后，可切 tail10 或 mix24 + train。
9. mix24 稳定后切 full24 + eval/stress 做泛化测试。
```
